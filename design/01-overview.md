# Overview

Vobes is a developer command center.

Think somewhere between:

- project manager
- Git dashboard
- developer workspace
- local knowledge base
- intelligent project assistant

Today, developer context is fragmented across:

- Git status
- recent commits
- project health
- package managers
- frameworks
- local notes
- TODOs
- build status
- project metadata
- AI context

Vobes unifies all of this into a single place. Open Vobes, immediately
understand:

- what projects exist
- what has changed
- what needs attention
- what you were working on
- where to continue

## One System, Two Faces

The **CLI** (`vbs`) and **Desktop** (Vobes) are not separate applications.
They are two interfaces to the same system, sharing one core.

## Definitions

| Term | Meaning |
|---|---|
| Vobes | The product — developer command center |
| Vobe | One software project managed by Vobes |
| `vbs` | The CLI binary |
| Vobes | The desktop app |
| Activity | Any tracked event about a vobe (opened, modified, committed…) |

## Goal

A developer should open Vobes and think:

> "I want to manage all of my projects here."
