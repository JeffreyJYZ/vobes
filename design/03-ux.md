# User Experience

The product should feel:

- minimal
- calm
- fast
- modern
- native
- keyboard-friendly
- developer-focused

## Rules

### Avoid Clutter

- No unnecessary panels.
- No decorative noise.
- Information density on demand, not always on.

### Prefer Excellent Defaults

- Sensible scan roots.
- Sensible display format.
- Sensible shortcuts.

The user should not need to configure anything to get a great experience.

### Keyboard-First

- Every common action must have a shortcut.
- Mouse optional.
- Discoverable: help menu always available.

### Native Feel

- Respect OS conventions.
- Match system theme when possible.
- No web-app-looking electron vibes.

### Speed

- Instant startup.
- Instant scan for small workspaces.
- No spinners for cached data.

## Pillars

| Pillar | Implementation |
|---|---|
| Minimal | Defaults work, optional everything |
| Calm | No color overuse, no notifications spam |
| Fast | Rust, no startup penalty, incremental updates |
| Native | Tauri, system fonts, system theme |
| Dev-focused | Git-first, terminal-adjacent, scriptable |
