# Product Ethos

> Quality over quantity.

## Principles

### Favor Quality Over Quantity

A small, sharp product beats a sprawling one.

### Cut Features, Never Architecture or UX

When forced to choose:

1. Remove the feature.
2. Keep the architecture clean.
3. Keep the user experience excellent.

Never compromise on arch or UX to ship more features.

### "I want to manage all of my projects here."

Every design decision is evaluated against this feeling.

- Does this help a developer centralize their work?
- Does this reduce friction?
- Does this respect the developer's attention?

### Modular by Default

Code should be divided into reusable modules with clean boundaries.
But no enterprise complexity — boundaries exist to serve clarity, not ceremony.

### Surfaces, Not Replacements

Vobes is a **surface** for information, not a replacement for tools.

- Git stays Git. We show its state.
- Editors stay editors. We launch them.
- Package managers stay package managers. We just identify them.

### Calm

The product should never feel busy, noisy, or anxious.

See [UX](./03-ux.md).
